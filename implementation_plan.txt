# Hedef: SQL Sorgularını Veritabanına (Stored Procedure) Taşımak

Kullanıcının isteği doğrultusunda `backend.py` dosyasında bulunan tüm ham SQL (raw SQL) sorgularını veritabanına taşıyacağız. Veritabanında **Stored Procedure (Saklı Yordam)** yapıları oluşturulacak ve `backend.py` sadece bu prosedürleri çağırarak (bridge görevi görerek) çalışacak.

## User Review Required

> [!IMPORTANT]
> **Dinamik Sorgular (Arama ve Sıralama):** `GET` isteklerinde kullanılan dinamik arama (search) ve sıralama (sort_by, order) işlemleri için Stored Procedure'ler içerisinde `PREPARE STATEMENT` yapısı kullanılacaktır. Bu durum, prosedürlerin biraz daha karmaşık görünmesine sebep olsa da tüm iş yükünü MySQL tarafına aktarmamızı sağlayacaktır.
> **Güvenlik:** Dinamik sıralamada SQL Injection riskini engellemek için prosedür içerisinde izin verilen sütun adları beyaz liste (whitelist) veya mantıksal kontrollerle denetlenecektir.

---

### Backend Değişiklikleri (`backend.py`)

#### [MODIFY] [backend.py](file:///d:/projects/Proje/backend.py)
Tüm rotalardaki (`/api/...`) string formatındaki SQL sorguları tamamen kaldırılacak. Bunun yerine `cursor.callproc('ProsedurAdi', [parametreler])` yapısı kullanılacaktır. Veya basit argüman geçişleri için `CALL ProsedurAdi(...)` kullanılacaktır.

Örnek dönüşüm:
```python
# ESKİ:
query = "INSERT INTO Urunler (Urun_Adi, Barkod, ...) VALUES (%s, %s, ...)"
cursor.execute(query, values)

# YENİ:
cursor.callproc('AddUrun', [Urun_Adi, Barkod, ...])
```
